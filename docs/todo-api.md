# Todo API Specification

## Product Scope

The backend manages a user's todo items and supports the core workflow used by
the React app: list tasks, add a task, mark a task completed or incomplete, and
delete a task.

## Data Model

```ts
type Todo = {
  id: string;
  title: string;
  completed: boolean;
  createdAt: string;
  updatedAt: string;
};
```

Validation rules:

- `title` is required after trimming whitespace.
- `title` length must be between 1 and 160 characters.
- `completed` defaults to `false`.
- `id` is server-generated.
- Timestamps are ISO 8601 strings generated by the server.

## Endpoints

### `GET /api/todos`

Returns all todo items for the current user, newest first.

Response `200`:

```json
{
  "data": [
    {
      "id": "todo_123",
      "title": "Review API endpoint design",
      "completed": false,
      "createdAt": "2026-06-21T09:00:00.000Z",
      "updatedAt": "2026-06-21T09:00:00.000Z"
    }
  ]
}
```

### `POST /api/todos`

Creates a todo item.

Request body:

```json
{
  "title": "Ship the todo list UI"
}
```

Response `201`:

```json
{
  "data": {
    "id": "todo_124",
    "title": "Ship the todo list UI",
    "completed": false,
    "createdAt": "2026-06-21T09:05:00.000Z",
    "updatedAt": "2026-06-21T09:05:00.000Z"
  }
}
```

### `PATCH /api/todos/:id`

Updates mutable todo fields. The app currently uses this endpoint to toggle
completion status, but it can also support title editing later.

Request body:

```json
{
  "completed": true
}
```

Response `200`:

```json
{
  "data": {
    "id": "todo_124",
    "title": "Ship the todo list UI",
    "completed": true,
    "createdAt": "2026-06-21T09:05:00.000Z",
    "updatedAt": "2026-06-21T09:08:00.000Z"
  }
}
```

### `DELETE /api/todos/:id`

Deletes a todo item.

Response `204`: no response body.

## Error Format

All non-`204` errors use a consistent JSON envelope.

```json
{
  "error": {
    "code": "VALIDATION_ERROR",
    "message": "Title is required."
  }
}
```

Expected status codes:

- `400` for malformed JSON or unsupported field types.
- `404` when `:id` does not match an existing todo.
- `422` when a valid request body fails domain validation.
- `500` for unexpected server failures.

## Backlog

- Persist todos in a database table keyed by `userId`.
- Add authentication before exposing multi-user data.
- Add pagination or cursor support if task volume grows.
- Add optional `PUT /api/todos/reorder` if manual ordering becomes required.
